# 한국 공공데이터 활용도 TOP10 + 1인 개발자용 비즈니스 아이디어 5건

> 작성일: 2026-04-29
> 원 프롬프트: `_docs/prompts/20260429_first_prompt.md`
> 검증 방식: 데이터셋 URL/License/제공형식은 공공데이터포털·각 운영기관 페이지로 직접 확인. 누적 활용신청 수치는 공식 집계가 dataset 페이지 단위로만 노출되어 정확 비교가 어려워, **명시된 수치는 모두 "추정" 표기**.

## TL;DR (3줄)
- 한국 공공데이터 중 **상업이용까지 안전한 KOGL 1유형 + 안정 운영** 데이터셋은 의외로 좁다 — 본 표의 10개가 실질적인 "기반 자원"이다.
- 1인 개발자 6개월 MVP 관점에서 가장 유망한 영역은 **B2B SaaS (거래처 모니터링·정부지원사업 매칭)** — 데이터가 정형이고, 페인이 명확하며, 유료 전환이 쉽다.
- B2C는 이미 포화구간이 많아 **수직 니치 + 모니터링·알림 자동화**로만 차별화 가능 (예: 어린이집 대기, 의약품 회수 매칭).

## Step 2. 비즈니스 아이디어 5건

### 아이디어 1. **거래처 휴폐업·세금계산서 리스크 모니터링 SaaS** ("BizGuard")

1. **One-liner**: **자영업자·중소기업 경리/대표**에게 **거래처가 폐업·간이→일반과세 전환 시 세금계산서 사고 발생** 위험을, **거래처 일괄 등록 + 매일 자동 모니터링·알림**으로 막는다.
2. **사용 데이터셋**:
   - **메인**: 국세청 사업자등록 진위확인·상태조회 API (Step 1 표 #4)
   - **결합 (외부)**: 토스/뱅크샐러드 사업자 입출금 명세 (제휴), 자체 OCR로 세금계산서 파싱 (v2 옵션)
3. **타겟 사용자 & 페르소나**:
   - 1차: **연매출 10~100억 중소제조/도매업 경리담당자** (거래처 50~500개 보유, 엑셀로 관리 중, 매년 1~2회는 "거래처 폐업으로 매입세액 불공제" 사고 경험)
   - 2차: **개인 세무사/회계사** (고객사 거래처 일괄 모니터링 도구)
4. **수익모델 (구체 수치)**:
   - SaaS 구독: 거래처 100개까지 **월 19,900원**, 500개 월 49,900원, 1,000개+ 월 99,000원
   - 가정: ARPU 월 35,000원, 6개월 100 유료고객 = MRR 350만원, 1년 500고객 = MRR 1,750만원
   - 추가: 세무회계 SaaS(자비스앤빌런즈/택소노미)와 API 제휴, 좌석당 5,000원/월 분배
5. **차별화 포인트 (moat)**:
   - 단일 조회는 토스/세무프로그램에 다 있음. **차별화는 "변경 발생 시점 즉시 알림" + "거래처 50~500개 일괄 import + 변경 이력 타임라인"**
   - 데이터 자체는 무료지만, **1일 100만 호출 제한 내에서 거래처 그룹별 차등 폴링 스케줄링**과 변경 이력 DB가 진짜 자산이 된다 (시간 갈수록 신규 진입자가 못 따라옴)
   - 세무 LLM 추가 (계산서 발행 가능 여부 실시간 판정) → 단순 알림 → 의사결정 지원으로 격상
6. **기술 난이도 & MVP 개발 기간**: **3개월 (LOW-MEDIUM)**
   - 백엔드: FastAPI + PostgreSQL + 일 1회 배치 폴링
   - 프론트: Next.js (대시보드 1개 + 거래처 목록 + 변경이력)
   - 결제: 토스페이먼츠 정기결제
7. **리스크**:
   - 국세청 API의 1일 100만건 제한 — 거래처 1만개 고객 100명 시 일 100만건 → 한도 빠르게 도달. 호출 분산·캐싱·증액신청 필요.
   - 토스가 "비즈머니 거래처 관리"를 강화하면 직접 경쟁 (실제 일부 기능은 이미 존재)
   - 폐업/사업자 상태가 NTS 전산 반영 지연 시 false negative 가능 → SLA 명시 필수

### 아이디어 2. **의약품 회수 알림 + 본인 복용약 자동 매칭** ("RecallMate")

1. **One-liner**: **만성질환자·고령자 가족**에게 **식약처가 회수·판매중지를 발표한 의약품이 본인 복용약과 일치할 경우 즉시 알림**으로 위해 사고를 차단한다.
2. **사용 데이터셋**:
   - 식약처 의약품 안전정보 + 회수정보 (7번)
   - 결합: HIRA 약국 위치 (8번, 환불·대체 안내), 식약처 e약은요 사용자 수기 등록, 처방전 OCR (옵션, 의료법 검토 필요)
3. **타겟 사용자 & 페르소나**:
   - 1차: **60대 부모님과 떨어져 사는 30~40대 자녀** (부모 복용약 5~10종, 회수 뉴스 봐도 본인 약인지 확인 어려움)
   - 2차: **요양원·노인주간보호센터 시설장** (입소자 30명 평균 복용약 8종, 회수 발생 시 수동 대조)
4. **수익모델 (구체 수치)**:
   - B2C 무료 + Pro 월 2,900원 (가족계정 5인, 처방전 사진 보관, 복약 알림)
   - B2B SaaS: 요양원 좌석당 월 9,900원 × 시설별 30~50석. 전국 등록 요양시설 약 5,800개 — 1% 침투 = 58개 시설 × 평균 40석 × 9,900 = 월 매출 2,300만원
   - 광고 X (안전 신뢰 도메인이라 광고 BM 부적합)
5. **차별화 포인트 (moat)**:
   - 식약처는 '의약품안전나라'에서 회수 공시는 하지만 **개인 복약과의 자동 매칭은 안 함**. 이게 핵심 갭.
   - 약학정보원·메디게이트는 의료진 대상이라 일반인 UX 부재
   - **시설용(B2B) 입소자 DB 구축**이 lock-in. 신규 진입자가 따라잡으려면 다시 입력해야 함.
6. **기술 난이도 & MVP 개발 기간**: **5개월 (MEDIUM)**
   - 회수 정보 매칭은 의약품 표준코드(KD코드/EDI코드) 기반 → 정형 매칭으로 단순
   - 처방전 OCR은 v2로 분리 (의료법 자문 + 정확도 검증 필요)
   - Flutter 모바일 + FastAPI
7. **리스크**:
   - 처방전 OCR이 마케팅 핵심인데 **개인정보보호법 + 약사법 + 보험사기방지법** 검토 필요. 잘못하면 의료데이터 처리 위반.
   - 회수 사건 자체가 자주 일어나지 않아 retention이 약할 수 있음 → 복약 시간 알림 등 보조 기능으로 daily active 유지 필요
   - "굿닥 + 약학정보원"이 동일 기능 추가 시 빠르게 잠식

### 아이디어 3. **농산물 도매가 인텔리전스 SaaS for 식당/식자재 도매** ("MarketGreen")

1. **One-liner**: **외식업 사장·식자재 마트 발주담당**에게 **이번주 도매가 추세와 다음주 예상 변동**을 보여줘 **발주 타이밍·메뉴 단가 조정**을 돕는다.
2. **사용 데이터셋**:
   - KAMIS 농산물 도/소매 가격 (대체 데이터셋)
   - 결합: 기상청 단기예보(1번, 작황 영향), KOSIS 가공식품물가지수(10번)
3. **타겟 사용자 & 페르소나**:
   - 1차: **수도권 한식·분식 점주** (월매출 3~8천만원, 5~15가지 식자재 발주, 현재 가락시장 시세를 카톡·블로그로 수동 체크)
   - 2차: **지역 식자재 마트 발주담당자** (50~200개 SKU 매주 시세 모니터링)
4. **수익모델 (구체 수치)**:
   - SaaS: 점포당 **월 29,000원** (Lite, 식자재 10종) / 월 79,000원 (Pro, 식자재 50종 + 메뉴 원가율 알림)
   - 가정: 1년 차 200 점주 × 평균 4만원 = MRR 800만원
   - B2B 수직 확장: 프랜차이즈 본사 일괄 도입 (점포 100개당 연 600만원 패키지)
5. **차별화 포인트 (moat)**:
   - KAMIS 사이트는 무료지만 **점주가 매주 들어가서 보지 않는다** — 페인은 "비교지표 + 알림". 차별화 = **메뉴별 원가 임팩트 자동 계산** (배추 +20% → 김치찌개 원가 +650원 알림)
   - 데이터는 같지만 **점주의 메뉴/원가표를 등록받아 둔 시간만큼 lock-in**
   - 농수산식품유통공사가 자체 앱(가락시장)을 강화해도 메뉴 단위는 안 함
6. **기술 난이도 & MVP 개발 기간**: **3~4개월 (LOW)**
   - 데이터 수집 단순 (KAMIS API 일배치 + 자체 DB)
   - Next.js 단일 대시보드 + 모바일 알림
7. **리스크**:
   - 외식업주는 **IT 도입 의향과 결제율이 낮은 편** — 유료 전환 어려울 수 있음
   - 트래픽이 한정된 시장 (식당 60만 중 IT 활용 가능 점주 추정 5만 이하) — 시장 천장이 빠르게 옴
   - 차별화 핵심 "메뉴별 원가 자동 계산"이 단순 엑셀 매크로로 대체 가능해 보일 위험

### 아이디어 4. **중소기업 정부지원사업 매칭 + 신청 보조 SaaS** ("GovMatch")

1. **One-liner**: **연매출 10~300억 중소·소상공인 대표/총무**에게 **본인 회사가 신청 가능한 정부 지원사업·R&D 공고를 자동 큐레이션**하고 **공통 양식의 자동완성 + 마감 임박 알림**으로 신청 누락을 막는다.
2. **사용 데이터셋**:
   - 기업마당 정부지원사업 (대체 데이터셋, `bizinfo.go.kr`)
   - K-Startup, 지자체별 지원사업 공고 (스크래핑 — 비정형)
   - 결합: 국세청 사업자등록 정보(4번)로 업종 자동 입력, KOSIS 통계(10번)로 매출 규모/지역 매칭, NICE/KCB 신용 (옵션)
3. **타겟 사용자 & 페르소나**:
   - 1차: **연매출 50억 이하 제조·R&D 중소기업 대표** (지원사업 정보 부족, 컨설팅 업체에 건당 10~30% 성공보수 지급 중)
   - 2차: **지원사업 컨설턴트** (1인 컨설팅 사무소, 고객 30곳 일괄 모니터링 도구로 사용)
4. **수익모델 (구체 수치)**:
   - 베이직 (매칭만): **월 19,000원** / 프로 (서류 자동완성·이력 관리): 월 49,000원 / 컨설턴트 플랜: 좌석당 월 99,000원, 고객사 30개까지
   - 가정: 6개월 200 유료기업 + 30 컨설턴트 = MRR 약 (200×35,000) + (30×99,000) = 1,000만원
   - **+ 컨설턴트 매칭 리퍼럴 수수료 5%** (계약 발생 시) — Bizinfo의 약점 보완
5. **차별화 포인트 (moat)**:
   - bizinfo.go.kr 자체에 검색은 있지만 **본인 회사 프로필 기반 자동 큐레이션이 없음**, 또한 **지자체·민간 공고가 분산**돼 있어 통합이 가치
   - 신청 양식 자동완성 = 컨설턴트 노동의 80%를 자동화 → 양식 데이터셋이 시간 갈수록 자산화
   - **지원사업 채택 결과 데이터**(자체 수집)로 "실제 어떤 회사가 받았는가" 통계 → 매칭 정확도 격차 점진 확대
6. **기술 난이도 & MVP 개발 기간**: **5개월 (MEDIUM)**
   - 어려운 부분: 비정형 공고(PDF·HWP) 정제 — LLM(JSON 추출)으로 60% 자동화 + 운영자 보정 30분/일
   - 매칭 룰엔진: 업종코드/지역/매출/연차/직원수 — 정형 SQL로 충분
7. **리스크**:
   - **bizinfo.go.kr 자체 개선** 또는 **정부24 통합** — 위협적이나, 공공 서비스의 UX 개선 속도가 느려 1~2년 안전
   - 컨설턴트 시장의 정치적 견제 (수수료 모델 비판)
   - "이미 비즈매칭/비스마트 등 5~10개 유사 서비스 존재" — 이 중 사용성·자동완성으로 차별화 필수

### 아이디어 5. **어린이집·유치원 대기 자동 모니터링 + 평가 비교** ("KidsQueue")

1. **One-liner**: **수도권 만 0~5세 자녀를 둔 맞벌이 부모**에게 **반경 2km 어린이집의 빈자리/대기순번 변동을 매일 자동 체크하고 평가등급·실 통원 거리와 함께 비교**해 입소 전쟁의 의사결정 시간을 1/10로 줄인다.
2. **사용 데이터셋**:
   - 임신육아종합포털 어린이집 정보 (대체 데이터셋, `childcare.go.kr`)
   - 결합: 보건복지부 어린이집 평가 결과, KOSIS 영유아 통계(10번), 기상청 미세먼지(6번, 야외활동 가능 기관 선별)
3. **타겟 사용자 & 페르소나**:
   - 1차: **수도권 30대 맞벌이 첫째자녀 학부모** (육아휴직 복귀 6개월 전부터 매일 임신육아종합포털 수동 새로고침 중, 평균 30~50군데 비교)
   - 2차: **둘째 출산 직후의 1차 사용자** (재구매·LTV 확보)
4. **수익모델 (구체 수치)**:
   - Freemium: 무료 (반경 1km, 5개 기관 모니터링) / Pro 월 4,900원 (반경 5km, 무제한 모니터링, 빈자리 즉시 푸시)
   - 가정: MAU 5만 → 유료 전환 4% (육아 카테고리 평균치 추정) = 2,000명 × 4,900 = MRR 980만원
   - + 키즈 카테고리 광고 (제휴 어린이용품 커머스, 결제전환당 단가 3,000~10,000원)
   - + 맘카페·지역 커뮤니티 마케팅으로 CAC 5,000원 미만 가능
5. **차별화 포인트 (moat)**:
   - 임신육아종합포털 자체에 빈자리 조회는 있으나 **변동 모니터링·푸시·다중 기관 비교가 없음**
   - **평가등급 + 실거리(도보 통원시간) + 대기 변동을 한 카드에 묶은 비교 UX**가 진짜 가치
   - 사용자 등록 데이터(자녀 연령/희망 기관 list)가 LTV 자산. 둘째 출산 시 재활성화.
6. **기술 난이도 & MVP 개발 기간**: **3~4개월 (LOW)**
   - 데이터 정형 OK, 일 1회 폴링 + 변동 푸시
   - Flutter 모바일 + FastAPI + Firebase Cloud Messaging
7. **리스크**:
   - 임신육아종합포털 정책 변경(자체 모니터링 기능 추가) 시 가치 잠식 — 하지만 공공 UX 개선 속도 느려 1~2년 안전
   - **유료 전환 가격 천장이 낮음** — Pro 4,900원으로는 LTV가 낮을 수 있음, 광고 BM 비중을 높이는 전략 필요
   - 지역별 데이터 품질 편차 (대도시 OK, 중소도시 빈약)
